    AI Copilot for developers

    2. Composer — from requirements to app blueprint

    Composer is Appian’s AI-powered planning assistant. In 25.4, it becomes a first-class, generally available feature for turning requirement documents into a structured Appian application plan.

    Key capabilities in 25.4

    • General availability — Composer is no longer preview; it’s integrated as a platform feature.
    • Rich input formats — Upload TXT, DOC, DOCX, or PDF requirement files.
    • Automatic app plan — Composer parses the document into features, personas, and user stories.
    • Business rules suggestions — It identifies potential rules and lets you generate expression rules from them.
    • Process diagrams + models — Generates swimlane diagrams and starter process models with tasks and gateways.
    • Reports & dashboards — Can produce a “Reports & Dashboards Library” page connected to Process HQ.

    The net effect: less time manually reading long BRDs and more time refining the logic, data, and UX that actually matter.

    AI Agents

    3. Agent Studio — autonomous, process-aware agents

    Agent Studio lets you create AI agents that use your existing Appian objects to move work forward. Instead of hard-coding every path, you define the goals and let agents figure out the steps.

    • Describe an agent’s mission in natural language (e.g., “triage support emails and route to the right queue”).
    • Agents can call into your process models, leverage Data Fabric, and apply business rules.
    • They’re designed for open-ended work like case triage, dispute handling, or complex customer requests.
    • All activity happens with enterprise security, auditing, and governance in mind.
    AI in process

    4. AI Skills, model selection, and long-running calls

    AI inside process models becomes more controllable and robust, especially for long or complex calls.

    • Admins enable or disable generative models directly in the Admin Console.
    • AI Skills can target specific models from the configured list.
    • Execute Generative AI Skill supports:
      • Standard mode for fast responses.
      • Long-running mode for large or complex outputs.
    • Use a!genAiModels() to discover available models dynamically from UIs or utilities.
    • Override the configured model at runtime in the smart service for per-execution tuning.
    AI Copilot & chat

    5. AI Copilot, chat field, and conversational UX

    AI Copilot becomes more usable in actual apps with a dedicated chat field component and extended region availability.

    • New chat field UI component for building conversational experiences inside interfaces.
    • Use a!callLanguageModel() to send prompts and conversation history to Appian’s language model.
    • Perfect for helpdesk assistants, guided flows, or conversational document Q&A.
    • Enterprise Copilot and smart search expand availability to the Europe (Zurich) region.
    Data & search

    6. Data Fabric scale, TDE, and smarter search

    Data Fabric in 25.4 can handle more rows, more writes, and more secure storage, while smart search and document monitoring improve visibility.

    Scale & performance

    • Record types can sync up to 50 million rows.
    • Write throughput significantly improved (targeting up to ~30k tx/min in supported workloads).
    • Better concurrency support for high-volume transactional use cases.

    Security & resiliency

    • Support for transparent data encryption (TDE) for synced record data at rest.
    • Option to skip failed smart service syncs and keep using last-good data while you fix issues.

    Smart search & documents

    • Refined similarity scoring, easier to tune how broad or strict results should be.
    • Configurable tolerance for index failures, improving overall availability.
    • Indices update automatically on incremental syncs.
    • New Documents view shows storage usage and highlights unused files for cleanup.
    Process analytics

    7. Process HQ & Copilot-driven reporting

    Process HQ grows into a hub where AI, data fabric, and process mining come together for business-friendly reporting.

    • Create reports by describing them in natural language (“Show cases closed per month by region”).
    • AI suggests filters, chart types, and ranges, which you can tweak and save.
    • Reports and dashboards can be deployed with your apps and exposed in sites.
    • More powerful one-to-many reporting, duration filters, and high availability for process insights.
    Platform scale

    8. Autoscale & high-volume error handling

    Autoscale is about running massive workloads. In 25.4, error diagnostics and control for large process sets improve significantly.

    • Error dashboard groups failures by node and type to speed up diagnosis.
    • Fix issues in bulk from monitor mode (modify variables, skip or restart nodes).
    • Cancel queued processes when halting a model at scale, freeing up capacity.
    RPA

    9. Robotic Process Automation performance

    RPA is tuned for environments with many robots and tasks, making the console faster and easier to work with.

    • Console load times and robotic task views are significantly faster.
    • Filtering options align more closely with Appian Designer for consistency.
    Governance

    10. Control Panel & Case Management Studio

    Object management in Control Panel gets more granular, and Case Management Studio gains new modules.

    Control Panel

    • Control Panel hierarchy items are separate objects with versioning and deployment.
    • You can delete unused items to keep the panel tidy.
    • Reorder configuration items to design the workspace layout.

    Case Management Studio

    • All modules support Case Management Studio 2.0.
    • Case creation via email, including sending from a case and using templates.
    • Knowledge Base module for attaching reference content to case types.
    • Smart Search module for semantic search across cases and tasks.
    Design & UX

    11. Interfaces, sites, and overall UX polish

    Appian modernizes the interface designer and site experience, making daily design work faster and cleaner.

    Interface designer

    • New configuration pane with tabs (Data, Display, Style, etc.) and built-in search.
    • Better navigation and keyboard accessibility; updated color pickers and controls.
    • File upload component: previews, rename, download, and custom drop zone styling.
    • Async placeholders appear per component/column, not over the entire layout.

    Sites & portals

    • New header style with modern look and flexible page alignment.
    • Options to hide logo, profile, or settings links when needed.
    • Portals can integrate with Google Analytics or Adobe Analytics.
    • WalkMe can be enabled to guide users directly in sites/portals.
    Mobile

    12. Appian Mobile experience

    The mobile client focuses on smoother updates, better offline behavior, and cleaner resume experiences.

    • Branded full-screen prompt when a mobile app update is required.
    • Offline data sync automatically refreshes the current page when complete.
    • Record views now resume where the user left off after backgrounding the app.
    Administration

    13. Admin Console, users, and locales

    Admins get tighter control over AI, richer user data, and extended localization support.

    • Admin Console defines which AI models/profiles are available across the environment.
    • User record and Users page show locale, time zone, last login, and account status.
    • More language/locale options added for European markets and beyond.
    • New data-fabric.csv metrics log to analyze record usage at scale.
    • OAuth 2.0 refresh configuration can request scopes on refresh, supporting modern external APIs.
